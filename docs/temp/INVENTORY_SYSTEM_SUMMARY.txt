⚙️ CORE BUSINESS LOGIC
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Stock Formula:
  available = stockOnHand - reserved
  isLowStock = available <= lowStockThreshold
  isCritical = available === 0

Atomic Reservation (Prevent Overselling):
  Query: { variantSku, $expr: { $gte: [available, requestedQty] } }
  Update: { $inc: { reserved: qty } }
  Result: Returns null if condition fails → INSUFFICIENT_STOCK error

Reserve → Commit Flow (Successful Order):
  1. reserveStock(sku, 10, order-123)
     → reserved: 10, available: 90 (physical stock not touched)
  2. Payment succeeds
  3. commitReservedStock(inv, 10, order-123)
     → reserved: 0, stockOnHand: -10 (now = 90), available: 90
     → Audit: [{ action: 'reserve', qty: 10 }, { action: 'commit', qty: 10 }]

Reserve → Release Flow (Abandoned Cart):
  1. reserveStock(sku, 10, order-123)
     → reserved: 10, available: 90
  2. 30 minutes pass (no payment)
  3. releaseReservedStock(inv, 10, order-123)
     → reserved: 0, available: 100 (restored)
     → Audit: [{ action: 'reserve', qty: 10 }, { action: 'release', qty: 10 }]

Manual Adjustment (Return/Damage):
  adjustStock(inv, +10, "Return received", admin)
  → stockOnHand: 100 → 110
  → Audit: { action: 'adjust', qty: 10, metadata: { reason } }

Batch Management (FIFO Picking):
  addBatch(inv, { batchId: 'LOT-001', qty: 50, receivedAt: '2025-11-01' })
  → batches array: [LOT-001(50), LOT-002(30)] (oldest first)
  → getOldestBatch() → LOT-001 (use this for picking)
  → When committing stock, deduct from oldest batch first

🔄 ORDER WORKFLOW INTEGRATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1: Customer Creates Order
  → For each line item, call:
    await inventoryService.reserveStock(variantSku, qty, orderId, 'customer-id')
  → Order enters "PENDING" state
  → Customer has 30 minutes to complete payment

Step 2: Payment Processing
  → If payment succeeds:
    await inventoryService.commitReservedStock(inventoryId, qty, orderId, 'payment-gateway')
    → Order enters "CONFIRMED" state
    → Stock is permanently deducted

  → If payment fails or times out:
    await inventoryService.releaseReservedStock(inventoryId, qty, orderId, 'system')
    → Order enters "CANCELLED" state
    → Stock is restored for other customers

Step 3: Order Cancellation (any time before payment)
  → Call:
    await inventoryService.releaseReservedStock(inventoryId, qty, orderId, 'order-system')
  → Stock becomes available again

Step 4: Returns/Refunds
  → When processing return:
    await inventoryService.adjustStock(inventoryId, returnQty, 'Refund: KRT-001', 'refund-admin')
  → Stock is restored to inventory
