ARCHITECTURE HIGHLIGHTS
───────────────────────

1. STATUS PROGRESSION
   • Enforced at schema pre-save level
   • Cannot move backwards (except to cancelled before shipped)
   • COD orders auto-confirm
   • Timeline events auto-generated

2. TIMELINE AUDIT TRAIL
   • Append-only (immutable history)
   • 12 event types: order.*, payment.*, shipment.*, refund.*, return.*
   • Actor tracking: system, admin, customer
   • Metadata storage: amounts, reasons, references

3. INVENTORY INTEGRATION
   ┌─────────────────┐
   │  CREATE ORDER   │
   └────────┬────────┘
            │
    ┌───────v────────┐
    │ RESERVE STOCK  │  ← Atomic: prevents overselling
    └───────┬────────┘
            │
   ┌────────v─────────┐
   │ PAYMENT SUCCESS? │
   └────────┬─────────┘
       YES  │  NO
        ┌───v────┐      ┌──────────────┐
        │ COMMIT │      │ STAY PENDING │
        │ STOCK  │      │              │
        └────────┘      └──────────────┘
            │
   ┌────────v────────┐
   │  CANCEL ORDER?  │
   └────────┬────────┘
            │
   ┌────────v────────┐
   │ RELEASE STOCK   │  ← Frees inventory
   └─────────────────┘

4. PAYMENT FLOW
   • COD: Auto-confirm immediately
   • Card/UPI: Pending until payment.success webhook
   • Payment amount validated against order total
   • Prevents payment mismatches

5. CACHING STRATEGY
   • Order tracking endpoint: 30s cache
   • Reduces DB load on popular orders
   • Filters sensitive fields (payment ref)
   • Cache-Control headers set by API
